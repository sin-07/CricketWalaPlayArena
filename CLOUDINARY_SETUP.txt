CLOUDINARY SETUP GUIDE - Cricket Wala Play Arena
================================================

Cloudinary provides FREE image hosting and management:
- 25 GB Storage
- 25 GB Bandwidth per month
- Image transformations and optimization
- Perfect for gallery management

STEP 1: Sign Up for Cloudinary (FREE)
--------------------------------------
1. Go to: https://cloudinary.com/users/register/free
2. Fill in your details and create account
3. Verify your email address
4. Login to your Cloudinary dashboard

STEP 2: Get Your Credentials
-----------------------------
1. After logging in, go to: https://console.cloudinary.com/
2. You'll see your Dashboard with:
   - Cloud Name
   - API Key
   - API Secret
3. Copy these values

STEP 3: Add to .env.local File
-------------------------------
Create or update the .env.local file in your project root:

NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your_cloud_name_here
CLOUDINARY_API_KEY=your_api_key_here
CLOUDINARY_API_SECRET=your_api_secret_here

Example:
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=dcxyz123
CLOUDINARY_API_KEY=123456789012345
CLOUDINARY_API_SECRET=abcdefGHIJKLmnopQRSTuvWXYZ

STEP 4: Restart Your Development Server
----------------------------------------
After adding the environment variables:
1. Stop your dev server (Ctrl+C)
2. Run: npm run dev

STEP 5: Upload Images
----------------------
1. Login to admin panel: http://localhost:3000/admin/login
2. Go to Gallery Management
3. Click "Add New Image"
4. Select single or multiple images
5. Add title and description
6. Click "Upload Images"

Images will be automatically:
- Uploaded to Cloudinary
- Saved to MongoDB
- Displayed on home page "Our Cricket Facilities" section

FEATURES:
---------
✓ Single image upload
✓ Multiple images upload at once
✓ Edit image details
✓ Toggle visibility (active/inactive)
✓ Delete images
✓ Automatic optimization by Cloudinary
✓ Fast CDN delivery
✓ Images appear instantly on home page

TROUBLESHOOTING:
----------------
If upload fails:
1. Check if .env.local file exists and has correct values
2. Verify Cloudinary credentials are correct
3. Restart your dev server
4. Check browser console for errors
5. Make sure MongoDB is running

For more help: https://cloudinary.com/documentation
